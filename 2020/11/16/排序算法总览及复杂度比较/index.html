排序算法总览及复杂度比较

  • 排序算法分为内部排序和外部排序

内部排序：

  • 指将需要的所有数据都加载到内部存储器中进行排序

  • 包括：

    ​ 插入排序（直接插入排序、希尔排序）、选择排序（简单选择排序、堆排序）、交换排序（冒泡排序、快速排序）、归并排序、基数排序

外部排序：

  • 数据量过大，无法全部加载到内存中，需要借助外部存储进行排序

衡量一个算法的执行时间：

  • 事后统计的方法：在同一计算机相同状态下运行，比较哪个算法速度快
  • 事前估算的方法：通过分析时间复杂度来判断哪个更优

算法的时间复杂度：

时间频度：

​ 一个算法花费的时间与算法中语句的执行次数成正比，一个算法中的语句执行次数称为语句频度或时间频度，记为T（n），在估算时间复杂度的算式中可以忽略常数项和低次项，有些可忽略系数

计算时间复杂度（O（n^2））：

​ 1）先用常数1代替运行时间中所有加法常数 T（n）=2n^2+7n+6=> T(n) =2n^2+7n+1

​ 2）只保留最高阶项：=>T(n)=2n^2

​ 3) 去除最高阶项的系数 => T(n)=n^2

常见的时间复杂度：

  • 常数阶O（1）[没有循环等复杂结构] ；

  • 对数阶O（log2(n)）[例：while（i<n）{ i=i×2}//若为i×3则：log3(n)] ;

  • 线性阶O（n）[单纯的for循环] ;

  • 线性对数阶O（n*log2(n)）[将时间复杂度为对数阶的代码执行n遍] ;

  • 平方阶O(n^2) [两层for循环] ;

  • 立方阶O（n^3）;

  • k次方阶 O（n^k）;

  • 指数阶 O（2^n）;

平均和最坏时间复杂度：
  • 二者是否相等与算法有关
  • 最差时间复杂度：最坏情况的时间复杂度(其余与平均一样)：

shell:O(n^s) 1<s<2 ; 快速： O（n^2）;

  • 平均时间复杂度：所有可能的输入实例均以等概率情况下时，该算法的执行时间：

冒泡：O（n^2）; 交换：O（n^2）; 选择：O（n^2）; 插入：O（n^2）;

基数：O（logR(B)）; shell：O（nlog(n)）; 快速：O（nlog(n)）; 归并：O（nlog(n)）;

堆：O（nlog(n)）;

算法的时间复杂度：

  • 该算法所耗费的存储空间，它也是问题规模n的函数
  • 空间复杂度是对一个算法在运行过程中临时占用存储空间大小的量度，n较大时占用较多的存储单元
  • 在做算法分析时主要讨论的是时间复杂度，所以一些算法的本身就是空间换时间

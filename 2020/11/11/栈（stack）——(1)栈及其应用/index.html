栈（stack）——(1)栈及其应用

  • 出栈（pop）和入栈（push）
  • 先入后出的有序列表
  • 栈限制线性表中的元素的插入和删除只能在 同一端，允许插入和删除的一端。为变化的一端，称为栈顶（Top），另一端为固定的一端，称为栈底（Bottom）

应用场景：

  • 子程序的调用：在跳往子程序前，会先将下个指令的地址存到堆栈中，直到子程序执行完后再将地址2取出，以回到原来的程序中。
  • 处理递归调用：和子程序的调用类似，只是除了储存下一个指令的地址外，也将参数，区域变量等数据存入堆栈中。
  • 当处理表达式时：表达式的转换[中缀表达式转后缀表达式]与求值，如：计算式：[7×2×2-5+1-5+3-3]
  • 二叉树的遍历
  • 图形的深度优化（depth-first）搜索法

实现思路（用数组模拟）：

实现栈的思路：

  • 用数组来模拟栈
  • 定义一个top来表示栈顶，初始化为-1
  • 入栈操作：当有数据加入到栈时，top++; stack[top]=data;
  • 出栈操作：int value =stack[top]; top–; return value;

应用：计算表达式(中缀)：

  • 创建一个数栈，一个符号栈

  • 通过一个index值（索引），来遍历表达式：

    ​ 如果是数字先判断是否为多位数，之后入数栈；

    ​ 如果是符号：若当前符号栈为空就直接入符号栈；若符号栈有操作符就进行比较（如果当前操作符的优先级小于或等于栈中的操作符，就需要从数栈中pop出两个数，从符号栈pop出一个符号，进行运算，将结果入数栈，然后将当前的操作符如符号栈；如果当前操作符的优先级大于栈中的操作符，就直接入符号栈；）

  • 当表达式扫描完毕，就顺序的从数栈和符号栈中pop出相应的数和符号，并运行（前一个数（后出栈的）运算符在前，后一个数（先出栈的）在运算符后）

  • 最后数栈只有一个数字，就是表达式的结果
